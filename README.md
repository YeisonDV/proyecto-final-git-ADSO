¿Qué es Git?

Git es un sistema de control de versiones distribuido que permite gestionar y trackear cambios en el código fuente durante el desarrollo de software. Fue creado por Linus Torvalds en 2005.



Entre sus características principales encontramos que:



Permite trabajar de manera colaborativa con otros desarrolladores



Mantiene un historial completo de todos los cambios realizados



Facilita la reversión a versiones anteriores si es necesario



Permite crear ramas para desarrollar features sin afectar el código principal



Cada desarrollador tiene una copia completa del repositorio



Es rápido, eficiente y funciona tanto online como offline



Git se ha convertido en el estándar de la industria para el control de versiones y es fundamental en el flujo de trabajo de cualquier desarrollador moderno.



10 Comandos de Git Explicados

1\. git init

Inicializa un nuevo repositorio Git en un directorio, creando la estructura necesaria para el control de versiones.



2\. git add

Agrega archivos al área de staging, preparándolos para ser incluidos en el próximo commit.



3\. git commit

Guarda los cambios del área de staging en el historial del repositorio con un mensaje descriptivo.



4\. git status

Muestra el estado actual del repositorio, incluyendo archivos modificados, agregados o eliminados.



5\. git push

Sube los commits locales al repositorio remoto en GitHub u otro servicio.



6\. git pull

Descarga los cambios del repositorio remoto y los fusiona con la rama local actual.



7\. git branch

Permite crear, listar o eliminar ramas del repositorio.



8\. git checkout

Permite cambiar entre ramas o restaurar archivos a versiones anteriores.



9\. git merge

Fusiona los cambios de una rama con la rama actual.



10\. git clone

Crea una copia local de un repositorio remoto completo con todo su historial.



Flujo de Trabajo Recomendado

Sincronizar: Siempre comenzar con git pull para tener la versión más actual



Crear rama: Usar git checkout -b feature/nombre para nuevas funcionalidades



Desarrollar: Hacer commits frecuentes con mensajes descriptivos



Subir cambios: Usar git push -u origin nombre-rama para compartir el trabajo



Pull Request: Crear PR en GitHub para revisión y merge



Actualizar: Volver a main y hacer git pull después del merge
